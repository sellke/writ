# Git-Notes Audit Channel Format

## Purpose

The **git-notes audit channel** is a git-native, immutable audit trail for Writ.
When a spec lands via `/ship`, Writ composes a condensed **audit digest** from the
spec's per-story "What Was Built" (WWB) records and attaches it — under a dedicated
`refs/notes/writ` ref — to the commit that actually lands on the base branch.
`/release` attaches a version-level rollup note to the release/tag commit.

The audit then lives in git, travels with the commit (once sync is configured),
survives file moves and squash-merges, stays out of the diff, and is queryable with
`git log --notes=writ`.

This channel is the **audit** counterpart to the **forward-looking** WWB record.
See the [WWB ↔ notes boundary](#wwb--notes-boundary) below and
[ADR-017](../decision-records/adr-017-git-notes-audit-channel.md) for the full
rationale.

## Notes Ref Convention

- **Ref:** `refs/notes/writ` (dedicated). Every operation passes `--ref=writ`
  explicitly.
- **Never** write to `refs/notes/commits` (the git default) — that would clobber a
  user's own notes.
- **Attach:** `git notes --ref=writ add -f -F <tmpfile> <sha>` (`-f` overwrites when
  re-shipping the same SHA — a re-ship supersedes the prior digest).
- **Read:** `git log --notes=writ` or `git notes --ref=writ show <sha>`.

## Spec-Level Digest Schema (attached at `/ship`)

Composed from each completed story's `## What Was Built` record in the spec folder,
plus the landed commit range.

```
Writ Audit Digest (spec)
========================
Spec:     {spec-folder-id} — {spec title}
Landed:   {base-branch} @ {landed-sha}
Source:   {source-range, e.g. abc123..def456} ({N} commits)
Shipped:  {YYYY-MM-DD} by {git user.name}

Stories
-------
| Story | Review | Drift | Coverage |
|-------|--------|-------|----------|
| story-1-... | PASS | None | 86% |
| story-2-... | PASS | Small (DEV-003) | 91% |

Aggregate
---------
Verdict:   {PASS | PASS w/ drift | DEGRADED}
Drift:     highest = {None|Small|Medium|Large}; DEV-IDs: {DEV-003, ...}
Coverage:  {aggregate/na}
Files:     {C} created, {M} modified
Review:    {total review iterations across stories}

Refs
----
spec:      .writ/specs/{spec-folder-id}/spec.md
drift-log: .writ/specs/{spec-folder-id}/drift-log.md (if present)

Generated by Writ {VERSION}
```

### Field provenance (every field traces to a WWB or git source)

| Field | Source |
|---|---|
| Spec id + title | Spec folder id; `spec.md` header title |
| Landed base + SHA | `/ship` land step (`git rev-parse <base>` after merge) |
| Source range + count | `git log <base>..<pre-land-tip>` (source commits squashed/merged) |
| Shipped date | `git`-derived date (`YYYY-MM-DD`) |
| Shipped by | `git config user.name` |
| Story rows | Per-story `## What Was Built` → `### Review Outcome` (Result, Drift), `### Test Results` (Coverage) |
| Aggregate verdict | Worst story `### Review Outcome → Result` |
| Aggregate drift | Highest `### Review Outcome → Drift` severity present |
| DEV-IDs | Union of `### Deviations from Spec` DEV-IDs across stories |
| Aggregate coverage | Aggregated/lowest `### Test Results → Coverage`; `na` when absent |
| Files C/M | Counts from each story's `### Files Created` / `### Files Modified` |
| Review iterations | Sum of `### Review Outcome → Iteration count` across stories |

**Composition rules:**

- Parse each `story-*.md` for its `## What Was Built` section only.
- Aggregate verdict = worst story result; drift = highest severity present;
  DEV-IDs = union.
- **Never** copy narrative "Implementation Decisions" prose verbatim if it risks
  leaking transcript-like content — include **only** the audit fields above.

### Fallback / nil input (minimal digest)

If no WWB records are found (e.g., a `--quick` spec, or manual work), attach a
minimal digest and log a warning:

```
Writ Audit Digest (spec, minimal)
=================================
Spec:     {spec-folder-id or "(none)"}
Landed:   {base} @ {landed-sha}
Files:    {git diff --stat summary}
Note:     No "What Was Built" records found — minimal digest.
Generated by Writ {VERSION}
```

## Version Rollup Schema (attached at `/release`)

Composed from the specs shipped since the previous release tag (reusing
`/release`'s existing changelog-from-completed-specs assembly — no re-scan).

```
Writ Release Audit — v{VERSION}
===============================
Released: {YYYY-MM-DD}
Tag:      {tag} @ {tag-target-sha}
Previous: v{prev-version}

Specs shipped this release
--------------------------
- {spec-folder-id} — {title} — {aggregate verdict, highest drift}
- ...

Changelog: CHANGELOG.md#{version-anchor}
Generated by Writ {VERSION}
```

The rollup **references** per-spec digests (from `/ship`) rather than duplicating
their full contents.

## Content Constraint (audit-only)

Digest content is **audit-only**: review verdict, coverage %, drift severity +
DEV-IDs, review iteration count, files touched, spec/story refs, landed SHA range.
It **never** includes chain-of-thought, prompts, or transcripts (Prime Directive
audit constraint). This is the same boundary the durable audit summaries in
[ADR-013](../decision-records/adr-013-recommended-autonomous-delivery.md) observe.

## Read Surface

- **Canonical command:** `git log --notes=writ` shows each digest inline under its
  landed commit; `git notes --ref=writ show <sha>` shows a single note.
- **`/status`:** prints one line when at least one Writ note exists:

  ```
  📝 Last audit note: {short-sha} — {spec title} ({date})
  ```

  Resolved via `git log --notes=writ -1` filtered to commits carrying a Writ note.
  If none exist, the line is omitted.
- **No bespoke reader command** — `git log --notes=writ` is canonical (excluded from
  scope).

## Sync Configuration

Git neither pushes nor fetches notes by default, so the audit must be sync-configured
or it stays local-only. `scripts/install.sh` (and platform setup) configures the
following refspecs on the default push remote (origin fallback), **unless opted out**:

```
git config --add remote.<remote>.fetch "+refs/notes/writ:refs/notes/writ"
git config --add remote.<remote>.push  "refs/notes/writ"
```

Sync setup is:

- **Idempotent** — existing refspecs are grepped first; no duplicates are ever added.
- **Guarded** — skipped entirely when the feature is opted out (see below).
- **No-op graceful** — when there is no git repo or no remote, it warns and skips
  (never fails install).

**Optional local hardening:** `git config notes.rewriteRef refs/notes/writ` makes
`git rebase` / `git commit --amend` copy Writ notes forward on *local* rewrites. It
does **not** rescue squash-merge (a new commit, not a rewrite), which is exactly why
the primary attach point is the post-land commit.

## Opt-Out (`writ.auditNotes`)

The single source of truth is a per-repo git config key, default `true`:

```bash
git config writ.auditNotes false     # disable the audit channel for this repo
git config --bool writ.auditNotes    # read it (commands treat absent as true)
```

Rationale for a git-config key: it is per-repo, travels in `.git/config`, and is
trivially checkable by `ship.md`, `release.md`, and `install.sh`.

When `writ.auditNotes` is `false`:

- `/ship` and `/release` skip note composition/attachment entirely (silent no-op).
- `install.sh` adds no refspecs and **removes any Writ-added `refs/notes/writ`
  refspecs** it previously created.
- Opt-out leaves **no residue**: no `refs/notes/writ` refspec lines, and no dangling
  config beyond the single `writ.auditNotes=false` marker.

## Non-Blocking Rule

Audit-note composition or attachment failure **never fails the ship or release**. It
logs a visible warning (`⚠️ audit note not attached — {error}`) and continues. This
mirrors the WWB principle: *"The pipeline must NEVER block completion due to
incomplete audit data. Partial records are better than no records."*

## WWB ↔ Notes Boundary

The audit channel and the [WWB record](what-was-built-format.md) are complementary,
not redundant:

| | What Was Built (WWB) | Git-notes audit digest |
|---|---|---|
| **When** | At each story completion | At `/ship` (spec digest) and `/release` (rollup) |
| **Where** | Inside the story markdown (working tree) | `refs/notes/writ`, bound to a commit SHA |
| **Mutability** | Editable after the fact | Immutable audit record, out of the diff |
| **Granularity** | Per-story, forward-looking context | Per-spec / per-version, backward-looking audit |
| **Audience** | Downstream agents (cross-story continuity) | Auditors / maintainers (`git log --notes=writ`) |

The digest **aggregates** WWB fields; it does not duplicate the full WWB narrative.
See [ADR-017](../decision-records/adr-017-git-notes-audit-channel.md) for the boundary
rationale.

## Non-Goals

Mirroring the spec's scope boundaries, this channel deliberately excludes:

- **Per-story branch-local notes** — dropped by squash-merge; the digest attaches to
  the surviving landed commit instead.
- **A bespoke `writ notes` reader command** — `git log --notes=writ` is canonical.
- **Retroactive backfill** of historical commits.
- **GitHub PR-comment mirroring** of the digest.
