# Story 2: `context.md` Artifact Map Schema

> **Status:** Not Started
> **Priority:** High
> **Dependencies:** Story 1
> **Story Points:** 3

## User Story

As an **agent or developer resolving where things live**, I want **an Artifact Map in the regenerated `context.md`**, so that **there's one auto-current place to find product/spec/docs artifacts and see integrity at a glance — without a separate index file.**

## Acceptance Criteria

1. **Given** the `context.md` schema, **when** it is regenerated, **then** it includes an `## Artifact Map` section (product, active spec, knowledge, docs) plus an `Integrity:` line.
2. **Given** missing artifacts, **when** the Map is generated, **then** it marks them and the Integrity line reports `⚠️ missing required: <list>`; when all present, `✅`.
3. **Given** regeneration, **when** it runs in `implement-story`, `implement-spec`, and `status`, **then** all three produce the Map (schema referenced from one canonical definition, not duplicated divergently).
4. **Given** absent optional sources, **when** the Map renders, **then** those sub-items are omitted (graceful degradation) while the Integrity line still renders.

## Implementation Tasks

- [ ] Extend the canonical `context.md` schema in `commands/implement-story.md` Step 2 with the `## Artifact Map` section (technical-spec §2).
- [ ] Update `commands/implement-spec.md` and `commands/status.md` regeneration references so they emit the Map (point to the canonical schema; avoid divergent copies).
- [ ] Define present-conditional rendering + the Integrity line logic.
- [ ] Ensure the Map is rewritten wholesale (never appended/patched), consistent with existing context.md regeneration discipline.

## Technical Notes

- `context.md` is regenerated by three commands — keep ONE canonical schema (in implement-story) and have the others reference it, matching how the existing schema is handled.
- The Integrity line is the cheap "handshake health check" equivalent to Conductor's link verification.
- See `sub-specs/technical-spec.md → §2`.

## Definition of Done

- [ ] Canonical schema updated with Artifact Map + Integrity line.
- [ ] implement-spec + status regeneration emit the Map.
- [ ] Graceful degradation verified (missing sources omitted, Integrity always renders).
